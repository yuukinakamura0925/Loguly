# 動画視聴ログ管理システム「Loguly」要件定義書

**バージョン:** 1.0
**作成日:** 2026-02-02
**ステータス:** ドラフト

---

## 1. プロジェクト概要

### 1.1 背景・目的

企業における研修動画やeラーニングコンテンツの視聴管理において、「従業員が本当に動画を視聴したか」を正確に把握することは困難である。単純な視聴履歴だけでは、早送りやスキップによる形式的な視聴完了を防ぐことができない。

本システム「Loguly」は、動画のスキップを制限し、実際の視聴時間を秒単位で記録することで、**確実な視聴完了を保証**する動画視聴ログ管理システムである。

### 1.2 システム概要

- ユーザーが動画を「スキップせずに」最後まで視聴したかを厳密に管理するWebアプリケーション
- 管理者は動画のアップロード、カテゴリ管理、ユーザーの視聴進捗をリアルタイムで把握可能
- クラウドベースのSaaSとして提供

### 1.3 対象ユーザー

| ユーザー種別 | 説明 |
|-------------|------|
| 一般ユーザー | 動画を視聴する従業員・受講者 |
| 管理者 | 動画・ユーザー・進捗を管理する研修担当者 |

### 1.4 用語定義

| 用語 | 定義 |
|------|------|
| 視聴完了 | 動画全体の90%以上を視聴した状態 |
| 最大視聴秒数 | ユーザーが視聴した最も先の再生位置（秒） |
| Heartbeat | 再生中に定期的にサーバーへ送信する視聴位置情報 |
| 署名付きURL | 有効期限・権限付きの動画アクセスURL |

## 3. 機能要件

### 3.1 ユーザー機能

#### 3.1.1 認証機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| U-AUTH-001 | ログイン | メールアドレス/パスワードでログイン | 必須 |
| U-AUTH-002 | ログアウト | セッション破棄してログアウト | 必須 |
| U-AUTH-003 | パスワードリセット | メールによるパスワード再設定 | 必須 |
| U-AUTH-004 | セッション維持 | ブラウザを閉じても7日間ログイン状態を維持 | 必須 |

**ビジネスルール:**
- アカウントは管理者のみが発行可能（セルフサインアップ不可）
- 連続5回のログイン失敗で15分間アカウントロック

#### 3.1.2 動画一覧機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| U-LIST-001 | カテゴリ別一覧表示 | 動画をカテゴリごとにグループ化して表示 | 必須 |
| U-LIST-002 | 視聴ステータス表示 | 各動画の視聴状態を視覚的に表示 | 必須 |
| U-LIST-003 | カテゴリ進捗表示 | カテゴリ単位での視聴完了率を表示 | 必須 |
| U-LIST-004 | 検索機能 | 動画タイトルでの検索 | 任意 |

**視聴ステータス定義:**

| ステータス | 条件 | 表示 |
|-----------|------|------|
| 未視聴 | view_log レコードなし | グレーアイコン |
| 視聴中 | 0% < 進捗 < 90% | 黄色アイコン + 進捗率 |
| 完了 | 進捗 >= 90% | 緑チェックマーク |

#### 3.1.3 動画プレイヤー機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| U-PLAY-001 | 動画再生 | Cloudflare Streamプレイヤーでの動画再生 | 必須 |
| U-PLAY-002 | スキップ制限 | 初回視聴時は最大視聴秒数より先へのシーク禁止 | 必須 |
| U-PLAY-003 | 巻き戻し許可 | 視聴済み区間への巻き戻しは常に許可 | 必須 |
| U-PLAY-004 | 自動再開 | 前回停止位置から自動再生開始 | 必須 |
| U-PLAY-005 | 進捗保存 | 5秒ごとにHeartbeat送信し進捗を保存 | 必須 |
| U-PLAY-006 | 完了検知 | 90%視聴で完了フラグをONに更新 | 必須 |
| U-PLAY-007 | 再生速度変更 | 0.5x〜2.0xの再生速度変更（完了後のみ） | 任意 |

**スキップ制限の詳細仕様:**

```
【初回視聴時（completed = false）】
  - シーク可能範囲: 0秒 〜 max_watched_seconds
  - max_watched_secondsより先へのシークは無効化
  - 巻き戻しは常に許可

【視聴完了後（completed = true）】
  - 全区間でシーク自由
  - 再生速度変更も許可
```

**Heartbeat仕様:**

```
送信間隔: 5秒ごと
送信データ:
  - user_id: ユーザーID
  - video_id: 動画ID
  - current_time: 現在の再生位置（秒）
  - timestamp: 送信時刻

サーバー側処理:
  - current_time > max_watched_seconds の場合のみ更新
  - current_time >= duration * 0.9 の場合、completed = true
```

#### 3.1.4 進捗管理機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| U-PROG-001 | 全体進捗表示 | ダッシュボードで全カテゴリの進捗率表示 | 必須 |
| U-PROG-002 | カテゴリ別進捗 | カテゴリごとの完了動画数/全動画数 | 必須 |

### 3.2 管理者機能

#### 3.2.1 動画管理機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| A-VID-001 | 動画アップロード | Cloudflare Stream APIへ動画をアップロード | 必須 |
| A-VID-002 | メタデータ設定 | タイトル、説明、カテゴリの設定 | 必須 |
| A-VID-003 | 表示順管理 | ドラッグ&ドロップで動画の表示順を変更 | 必須 |
| A-VID-004 | 動画削除 | 動画の削除（視聴ログは保持） | 必須 |
| A-VID-005 | 動画差し替え | 既存動画の映像ファイルのみ差し替え | 任意 |
| A-VID-006 | アップロード状況確認 | トランスコード完了までの進捗表示 | 必須 |

**動画アップロードフロー:**

```
1. 管理者が動画ファイルを選択（最大2GB）
2. Cloudflare Stream APIへダイレクトアップロード
3. Cloudflareがトランスコード処理を実行（WebHook通知）
4. トランスコード完了後、DBに動画情報を自動登録
5. 管理者がメタデータ（タイトル等）を設定
6. 公開設定をONにしてユーザーに公開
```

**対応フォーマット:**
- MP4, MOV, MKV, AVI, WebM
- 最大ファイルサイズ: 2GB
- 推奨解像度: 1920x1080 (1080p)

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| A-CAT-001 | カテゴリ作成 | 新規カテゴリの作成 | 必須 |
| A-CAT-002 | カテゴリ編集 | カテゴリ名の変更 | 必須 |
| A-CAT-003 | カテゴリ削除 | カテゴリの削除（動画が0件の場合のみ） | 必須 |
| A-CAT-004 | 表示順管理 | カテゴリの表示順を変更 | 必須 |

#### 3.2.3 ユーザー管理機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| A-USR-001 | ユーザー作成 | 新規ユーザーアカウントの作成 | 必須 |
| A-USR-002 | ユーザー編集 | ユーザー情報の編集 | 必須 |
| A-USR-003 | ユーザー削除 | ユーザーの論理削除 | 必須 |
| A-USR-004 | 一括登録 | CSVによるユーザー一括登録 | 任意 |
| A-USR-005 | パスワードリセット | 管理者による強制パスワードリセット | 必須 |

**ユーザー登録フロー:**

```
1. 管理者がユーザー情報を入力（氏名、メールアドレス）
2. 初期パスワードを自動生成
3. 招待メールを送信
4. ユーザーがメール内リンクからパスワードを設定
5. ログイン可能状態に
```


## 8. 外部連携

### 8.1 Cloudflare Stream連携

**アップロードフロー:**

```
1. フロントエンドから /api/admin/videos/upload-url を呼び出し
2. サーバーがCloudflare APIでダイレクトアップロードURLを取得
3. フロントエンドが取得したURLへ直接動画をアップロード
4. Cloudflareがトランスコード完了時にWebhookで通知
5. サーバーがWebhookを受信し、DBに動画情報を登録
```

**署名付きURL生成:**

```
1. ユーザーが動画詳細APIを呼び出し
2. サーバーがCloudflare APIで署名付きURLを生成（有効期限: 2時間）
3. 署名付きURLをレスポンスに含めて返却
4. フロントエンドが署名付きURLで動画を再生
```

### 8.2 Supabase連携

- **Auth:** ユーザー認証・セッション管理
- **Database:** PostgreSQLデータベース
- **Realtime:** 視聴状況のリアルタイム更新（管理画面）
